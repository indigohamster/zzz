# Design Rules

## 总原则
本项目采用“小步修改、持续验证”的开发方式。

## 禁止事项
- 禁止未经确认的大规模重构
- 禁止一次性同时修改多个核心系统
- 禁止删除已有调试日志，除非明确说明
- 禁止引入新的状态源导致系统不同步
- 禁止让武器视觉、攻击逻辑、判定框各走各的状态

## 武器系统规则
- 武器类型必须影响：
  - 攻击方式
  - 攻击距离
  - 判定框形状
  - 攻击动画/视觉反馈
  - 敌人克制关系
- 玩家朝向、武器朝向、攻击判定方向必须一致
- 不同武器不能只改名字，必须有真实手感差异

## 画板系统规则
- 画出来的类型必须被战斗系统实际消费
- 画板输出不应只停留在 UI 或文本层
- 识别结果需要进入武器生成/切换流程

## 地图生成规则
- 地图不能只是随机房间堆叠
- 应该包含节奏变化、风险收益、路线选择
- 地图生成修改必须避免破坏基本可通行性

## AI 协作规则
每个 AI 接手前必须先读：
1. PROJECT_STATE.md
2. AI_HANDOFF.md
3. DESIGN_RULES.md
4. BUG_LOG.md
5. TODO_QUEUE.md

每次完成后必须更新：
1. AI_HANDOFF.md
2. BUG_LOG.md，如本次涉及 bug
3. TODO_QUEUE.md，如完成或新增任务
