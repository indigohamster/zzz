# Codex Tasks

## 通用执行规则

每次执行任务前：
1. 先阅读 /docs/PROJECT_STATE.md
2. 再阅读 /docs/DESIGN_RULES.md
3. 再阅读 /docs/TODO_QUEUE.md
4. 再阅读 /docs/BUG_LOG.md
5. 最后阅读 /docs/AI_HANDOFF.md

每次执行任务时：
- 只处理一个明确问题
- 优先最小修改
- 不要大规模重构
- 不要同时改多个核心系统
- 保留必要调试日志
- 修改后写清楚测试方式

每次执行任务后：
- 更新 /docs/AI_HANDOFF.md
- 如涉及 bug，更新 /docs/BUG_LOG.md
- 如完成任务，更新 /docs/TODO_QUEUE.md

---

## 任务模板

```md
# Codex Task

## 目标
这里写本次要解决的问题。

## 背景
这里写为什么要做。

## 禁止
- 不要大规模重构
- 不要改无关系统
- 不要删除已有调试日志

## 必须检查
- 相关文件
- 相关函数
- 相关状态流

## 验收标准
- 标准 1
- 标准 2
- 标准 3

## 完成后必须输出
1. 修改了哪些文件
2. 根因是什么
3. 怎么测试
4. 是否有风险
5. 是否更新了 AI_HANDOFF.md
```

完成后请输出：

1. 是否成功创建 `/docs`

3. 每个文件的路径
4. 是否没有改动任何游戏逻辑代码
