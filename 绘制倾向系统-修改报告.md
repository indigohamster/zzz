# 绘制倾向系统 - 修改报告

> 修改时间：2026-06-15  
> 目标：让绘制内容真正影响战斗体验  
> 方法：基于 ShapeAnalyzer 指标计算绘制倾向，然后在战斗时应用倾向修正

---

## 一、修改概述

本次修改实现了"绘制倾向系统"，让玩家画出的不同图形真正影响战斗体验，而不仅仅是改变武器类型。

### 核心思路

1. **分析绘制图形** → 计算 4 种绘制倾向的强度
2. **存储倾向** → 在武器档案中添加 `drawingTendency` 字段
3. **应用修正** → 在战斗时根据倾向修正战斗属性

---

## 二、修改的文件

### 1. `ShapeAnalyzer.js` (新增功能)

**文件位置**：`src/game/ShapeAnalyzer.js`

**修改内容**：
- 新增 `calculateDrawingTendency(metrics)` 函数
- 新增 4 个辅助计算函数：
  - `calculateCircularScore(closedness, complexity, symmetry)`
  - `calculateSharpScore(pointedness, closedness, complexity)`
  - `calculateChaoticScore(complexity, closedness, symmetry)`
  - `calculateLongStraightScore(aspectRatio, complexity, pointedness)`

**功能说明**：
根据 ShapeAnalyzer 的现有指标 (`closedness`, `pointedness`, `complexity`, `aspectRatio`, `symmetryScore`) 计算 4 种绘制倾向的强度 (0-1)。

**计算规则**：
| 倾向 | 触发条件 | 效果 |
|--------|----------|------|
| 圆形 | `closedness` ≥ 0.6 | 高闭合度 |
| 尖锐 | `pointedness` ≥ 0.6 | 高尖锐度 |
| 混乱 | `complexity` ≥ 0.6 且 `closedness` 低 | 高复杂度 |
| 长直线 | `aspectRatio` ≥ 2.5 或 ≤ 0.4 且 `complexity` 低 | 极端宽高比 |

**返回值**：
```javascript
{
  type: "circular" | "sharp" | "chaotic" | "longStraight" | null,
  strength: number (0-1)
}
```

### 4. `AttackController.js` (应用倾向修正)

**文件位置**：`src/inkwell/AttackController.js`

**修改内容**：
- 新增 `applyDrawingTendencyModifiers(finalStats, weapon)` 函数
- `resolveBuildAttackStats` 函数现在会应用倾向修正

**修正规则**：

#### 圆形倾向 (circular)
| 属性 | 修正 | 说明 |
|------|------|------|
| `range` | +20% × strength | 更大范围 |
| `damage` | -15% × strength | 更低伤害 |
| `crit` / `criticalChance` | -10% × strength | 更低暴击 |
| `inspirationGainMultiplier` | +30% × strength | 更高灵感获取 |

#### 尖锐倾向 (sharp)
| 属性 | 修正 | 说明 |
|------|------|------|
| `crit` / `criticalChance` | +50% × strength | 更高暴击率 |
| `pierce` | +0-2 (基于 strength) | 更高穿透 |
| `range` | -15% × strength | 更小范围 |
| `damageVariance` | +0.15 × strength | 更高伤害浮动 |

#### 混乱倾向 (chaotic)
| 属性 | 修正 | 说明 |
|------|------|------|
| `attackSpeed` | +30% × strength | 更高攻速 |
| `useTime` | -23% × strength | 更快攻击 |
| `damageVariance` | +0.2 × strength | 更高伤害浮动 |
| `chaoticEffect` | true | 启用混乱效果 |
| `chaoticStrength` | strength | 混乱强度 |

**混乱效果** (在 `CombatSystem.js` 中实现)：
- 伤害随机乘数：0.7-1.3 (随 strength 增加更极端)

#### 长直线倾向 (longStraight)
| 属性 | 修正 | 说明 |
|------|------|------|
| `range` | +40% × strength | 更长距离 |
| `knockback` | +20% × strength | 更高击退 |
| `preferThrust` | strength > 0.5 | 偏好突刺攻击 |

## 三、系统设计说明

### 为什么这样可以让"绘制内容真正影响战斗"？

**修改前**：
- 玩家画圆形 → 武器类型变为 sword → 战斗体验 = 标准剑攻击
- 玩家画三角形 → 武器类型变为 sword → 战斗体验 = 标准剑攻击
- **问题**：虽然武器类型相同，但战斗体验没有明显差异

**修改后**：
- 玩家画圆形 → 武器类型 = sword + 圆形倾向 → 战斗体验 = 大范围、低伤害、高灵感获取
- 玩家画三角形 → 武器类型 = sword + 尖锐倾向 → 战斗体验 = 高暴击、高穿透、小范围
- **效果**：同样的核心武器类型，但战斗体验明显不同！

### 如何使用现有数据结构？

1. **ShapeAnalyzer** - 已有指标，只需新增计算函数
2. **WeaponProfile** - 新增 `drawingTendency` 字段，不修改现有字段
3. **AttackController** - 读取倾向并修正 `finalStats`，不影响现有架构
4. **CombatSystem** - 读取 `stats.chaoticEffect` 并应用随机效果

### 为什么不影响存档兼容？

- 新增的 `drawingTendency` 字段是 **可选字段**
- 旧存档没有这个字段 → `tendency` 为 `null` → 不应用任何修正 → 行为不变
- 新武器档案包含这个字段 → 应用倾向修正 → 新行为

## 四、测试方法

### 测试 1：验证倾向计算

**步骤**：
1. 启动游戏，进入 Studio 场景
2. 绘制不同类型的图形：

   - **三角形**：画一个尖锐的三角形
   - **混乱线条**：画混乱的涂鸦
   - **长直线**：画一条很长的直线
3. 完成绘制，查看控制台日志

**预期结果**：
```

[DrawingTendency] { type: "sharp", strength: 0.82 }
```

**步骤**：
1. 生成 4 种不同倾向的武器（相同武器类型，如都是 sword）
2. 在 Inkwell 场景中分别使用它们战斗

**预期结果**：
| 倾向 | 体验差异 |

| 圆形 | 攻击范围明显更大，但伤害更低 |

| 混乱 | 攻击速度更快，但伤害浮动大（有时高有时低） |
| 长直线 | 攻击距离更长，击退更强 |

## 五、已知限制与未来改进

### 当前限制

1. **长直线倾向的"突刺攻击偏好"未完全实现**
   - 当前只是设置 `preferThrust` 标志
   - 需要在 `AttackController.js` 中实际修改攻击模式

2. **圆形倾向的"灵感获取加成"未完全实现**
   - 当前只是设置 `inspirationGainMultiplier` 标志
   - 需要在灵感获取逻辑中实际应用这个乘数

3. **缺少 UI 反馈**
   - 玩家看不到自己武器的倾向类型和强度
   - 需要在武器结果面板或 HUD 中添加显示

### 未来改进建议

1. **实现突刺攻击偏好**
   - 在 `AttackController.js` 中，如果 `preferThrust` 为 true，将攻击模式改为 `thrust`

2. **实现灵感获取加成**
   - 在灵感获取逻辑中，读取 `inspirationGainMultiplier` 并应用

3. **添加 UI 反馈**
   - 在武器结果面板中显示倾向类型和强度
   - 在 HUD 中显示当前武器的倾向图标

4. **平衡性调整**
   - 根据实际测试调整修正系数（如 +20% range 可能太多或太少）
