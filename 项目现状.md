## 3. 原型目标

当前原型用于验证 Inkwell Deep 的第一版核心循环：

1. 在夜晚工作室绘制武器。
2. 根据绘制过程生成武器属性。
3. 进入类 Terraria 的墨水世界。
4. 在墨水世界中移动、跳跃、冲刺、挖掘、放置方块、战斗、开宝箱。
5. 击败夜晚 Boss。
6. Boss 死亡后出现返回传送门，由玩家选择何时离开。
7. 回到早晨，生成反馈、奖励与进度变化。

## 6. 关键文件说明

### 入口与启动

- `index.html`
  - 浏览器入口。
  - 包含主 canvas。
  - 加载 `src/bootstrap.js?v=19`。

- `styles.css`
  - 页面和 canvas 样式。
  - 使用 16:9 适配。
  - 开启 pixelated 渲染。

- `StartPrototype.ps1`

## 8. 当前风险与注意点

### 没有版本管理

当前目录没有 `.git`，无法查看提交历史、分支、diff 或回滚点。

建议尽快初始化 Git，并提交当前可运行状态：

```powershell
cd D:\InkwellDeep
git init
git add .
git commit -m "Capture standalone prototype"
```

### 没有自动化测试

当前没有发现测试脚本、构建脚本或包管理配置。

目前验证方式主要依赖浏览器手动运行。

### 没有构建系统

项目使用原生 ES module，通过 `index.html` 加载 `src/bootstrap.js`，再由 `bootstrap.js` 动态加载 `main.js`。

这适合快速原型，但随着文件和功能增多，后续可能需要考虑：

- 最小化引入 Vite 或其他轻量 dev server。
- 增加 lint / format。
- 增加基础 smoke test。
- 明确模块边界和导入关系。

### 核心逻辑文件开始变大

`AttackController.js`、`main.js`、`inkwell.js`、`WorldGen.js` 等文件已经比较大。

短期不一定需要立刻重构，但如果继续增加功能，建议优先避免把新逻辑继续塞进这些文件。
