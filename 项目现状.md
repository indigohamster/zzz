## 3. 原型目标

首次游玩：Chapter0 序章（租房房间探索 → 墨点出现 → 墨迹扩散 → 过渡到工作室）

后续循环：
1. 在夜晚工作室绘制武器
2. 根据绘制过程生成武器属性
3. 进入类 Terraria 的墨水世界
4. 移动/跳跃/冲刺/挖掘/放置方块/战斗/开宝箱
5. 击败夜晚 Boss
6. Boss 死亡后出现返回传送门
7. 回到早晨，生成反馈、奖励与进度变化

## 4. 当前场景流程

```
chapter0（序章）→ studio（绘制）→ inkwell（墨境）→ feedback（结算）→ 循环
```

## 8. 当前风险与注意点

### 版本管理
Git 已初始化，但 Chapter0 集成和设计文档尚未提交。

### 没有自动化测试
验证方式主要依赖浏览器手动运行。

### 没有构建系统
项目使用原生 ES module，通过 index.html → bootstrap.js → main.js 加载。

### 核心逻辑文件较大
`AttackController.js` (21348 bytes)、`Chapter0.js` (17428 bytes)、`main.js` (15500+ bytes) 需要关注模块边界。
