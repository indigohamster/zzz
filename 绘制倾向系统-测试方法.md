## 二、功能测试

### 测试 1：验证倾向计算

**目标**：验证 `ShapeAnalyzer.calculateDrawingTendency` 是否正确计算绘制倾向。

**步骤**：
1. 启动游戏，进入 **Studio 场景**（绘制武器场景）
2. 使用鼠标绘制以下图形：
   - **测试 1A**：画一个**闭合的圆形**
   - **测试 1B**：画一个**尖锐的三角形**
   - **测试 1C**：画**混乱的涂鸦**（很多杂乱线条）
   - **测试 1D**：画一条**很长的直线**
3. 点击"完成绘制"按钮
4. 查看浏览器控制台日志

**预期结果**：

**测试 1A - 圆形**：
```
[DrawingTendency] { type: "circular", strength: 0.6-1.0 }
[WeaponRecognized] { rawType: "sword", weaponType: "sword", archetypeId: "sword", ... }
```

**测试 1B - 尖锐**：
```
[DrawingTendency] { type: "sharp", strength: 0.6-1.0 }
[WeaponRecognized] { rawType: "sword", weaponType: "sword", archetypeId: "sword", ... }
```

**测试 1C - 混乱**：
```
[DrawingTendency] { type: "chaotic", strength: 0.6-1.0 }
[WeaponRecognized] { rawType: "sword", weaponType: "sword", archetypeId: "sword", ... }
```

**测试 1D - 长直线**：
```
[DrawingTendency] { type: "longStraight", strength: 0.6-1.0 }
[WeaponRecognized] { rawType: "spear", weaponType: "spear", archetypeId: "spear", ... }
```

**注意**：长直线可能会改变武器类型（因为 `aspectRatio` 极端），这是正常行为。

### 测试 3：验证战斗属性修正

**目标**：验证战斗时 `AttackController.resolveBuildAttackStats` 会应用倾向修正。

**步骤**：
1. 生成一个带有明确倾向的武器（如尖锐倾向，strength > 0.5）
2. 进入 **Inkwell 场景**（墨境战斗场景）
3. 开始战斗，查看控制台日志

**预期结果**：
```
[DrawingTendencyModifier] {
  type: "sharp",
  strength: 0.82,
  before: { damage: 12, range: 56, attackSpeed: 3.75, crit: 0.04 },
  after: { damage: 12, range: 48, attackSpeed: 3.75, crit: 0.06 }
}
```

**说明**：
- `crit` 从 0.04 提升到 0.06（因为尖锐倾向 +50% × 0.82）
- `range` 从 56 降低到 48（因为尖锐倾向 -15% × 0.82）

---

### 测试 4：验证战斗体验差异（核心测试）

**目标**：验证不同倾向的武器在战斗中的体验明显不同。

**步骤**：
1. 生成 4 个武器，每个都有不同的绘制倾向（但最好是相同的武器类型，如都是 sword）
2. 进入 Inkwell 场景，分别装备这 4 个武器
3. 与同一个敌人战斗，观察体验差异

**预期结果**：

| 武器倾向 | 战斗体验 |
|----------|----------|
| **圆形** | • 攻击范围明显更大（可以看到攻击弧更大）<br>• 伤害数字略低<br>• 暴击率略低 |
| **尖锐** | • 暴击率更高（可以看到更多黄色暴击数字）<br>• 攻击范围略小<br>• 伤害浮动更大 |
| **混乱** | • 攻击速度更快（可以感觉到攻击间隔更短）<br>• 伤害数字浮动大（有时 5，有时 15）<br>• 控制台会输出 `[ChaoticEffect]` 日志 |
| **长直线** | • 攻击距离更长（可以站在更远的地方攻击）<br>• 击退效果更强（敌人被击退更远） |

---

### 测试 5：验证混乱效果的随机性

**目标**：验证混乱倾向会产生随机伤害。

**步骤**：
1. 生成一个带有混乱倾向的武器（strength > 0.5）
2. 进入 Inkwell 场景
3. 多次攻击同一个敌人（至少 10 次）
4. 记录伤害数字

**预期结果**：
- 伤害数字应该在较大范围内浮动
- 例如：基础伤害 12，实际伤害可能在 8-16 之间
- 控制台应该输出 `[ChaoticEffect]` 日志，显示随机乘数

**示例日志**：
```
[ChaoticEffect] { chaoticMultiplier: 0.85, strength: 0.68 }
[ChaoticEffect] { chaoticMultiplier: 1.12, strength: 0.68 }
[ChaoticEffect] { chaoticMultiplier: 0.72, strength: 0.68 }
```

### 测试 6：验证存档兼容性

**目标**：验证旧版本生成的武器档案在新版本中仍然可以正常使用。

**步骤**：
1. 使用旧版本（没有绘制倾向系统）生成一个武器档案
2. 将武器档案保存到 localStorage 或手动创建 JSON 对象
3. 在新版本中加载这个武器档案
4. 进入战斗

**预期的武器档案（旧版本）**：
```javascript
{
  id: "weapon-1234567890",
  name: "Sword",
  archetype: {...},
  // 注意：没有 drawingTendency 字段
  finalStats: {...},
  ...
}
```

**预期结果**：
- 武器应该正常工作
- 没有倾向修正 applied（因为 `drawingTendency` 为 `undefined`）
- 控制台没有 `[DrawingTendencyModifier]` 日志

## 四、边界情况测试

### 测试 8：验证无明确倾向的情况

**目标**：验证当绘制图形没有明确的倾向时，系统会正确处理。

**步骤**：
1. 绘制一个**中等复杂度的图形**（既不是明显的圆形，也不是明显的三角形）
2. 完成绘制，查看控制台日志

**预期结果**：
- 如果所有倾向的得分都 < 0.3，则 `calculateDrawingTendency` 返回 `{ type: null, strength: 0 }`
- 控制台**没有** `[DrawingTendency]` 日志（或显示 `type: null`）
- 战斗时没有倾向修正 applied

---

### 测试 9：验证极端指标的情况

**目标**：验证当绘制指标极端时，系统不会崩溃。

**步骤**：
1. 绘制一个**完美的圆形**（非常高 `closedness`）
2. 绘制一个**非常尖锐的三角形**（非常高 `pointedness`）
3. 绘制**非常混乱的涂鸦**（非常高 `complexity`）
4. 绘制一条**非常长的直线**（非常极端 `aspectRatio`）

**预期结果**：
- 所有情况都应该正常处理
- `strength` 应该在 0-1 范围内（不会超过 1）
- 不会有 JavaScript 错误

## 八、常见问题排查

### 问题 1：没有 `[DrawingTendency]` 日志

**可能原因**：
- 绘制的图形没有明确的倾向（所有得分 < 0.3）
- `calculateDrawingTendency` 函数没有被调用

**解决方法**：
- 尝试绘制更极端的图形（完美圆形、非常尖锐的三角形等）
- 在 `WeaponGenerator.js` 中添加调试日志

---

### 问题 2：战斗体验没有明显差异

**可能原因**：
- 倾向强度 (`strength`) 太低
- 修正系数太小

**解决方法**：
- 绘制更极端的图形以提高 `strength`
- 调整 `AttackController.js` 中的修正系数（如将 +20% 改为 +30%）

---

### 问题 3：控制台报错

**可能原因**：
- 代码修改有误
- 函数导入错误

**解决方法**：
- 检查浏览器控制台的错误信息
- 确认所有修改后的文件都保存了
- 清除浏览器缓存并重新加载
